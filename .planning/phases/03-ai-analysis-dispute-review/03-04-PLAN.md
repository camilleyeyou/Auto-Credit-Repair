    - Filter items: `filteredItems = activeBureau === "all" ? items : items.filter(i => i.bureau === activeBureau)`

    - Page heading: "Dispute Review"

    - Bottom CTA: "Generate Letters for Approved Items" button — navigates to `/letters` (Phase 4 route, not yet active; use `<Link href="/letters">`)

    - Creditor name (heading)
    - Item type badge (e.g. "Late Payment", "Collection" — capitalize/humanize the enum value)
    - AI dispute reason paragraph — use hedged language as returned by Claude; display as-is
    - FCRA section badge: `§ {item.fcraSection}` with `title={item.fcraSectionTitle}` tooltip on hover (HTML `title` attribute is sufficient — no custom tooltip component needed)
    - Confidence indicator: display as percentage `{Math.round(item.aiConfidence * 100)}%` with a colored label (green ≥70%, yellow 40-69%, red <40%)

    - Skip button: disabled if `item.status !== "pending_review"`, gray/outline variant, calls `updateStatus({ disputeId: item._id, status: "skipped" })`
    - Approved/Skipped state badge visible when `item.status !== "pending_review"` (replace buttons with a status badge)

    **Optimistic update for updateDisputeStatus:**
    ```typescript
    const updateStatus = useMutation(api.disputeItems.updateDisputeStatus).withOptimisticUpdate(
      (localStore, args) => {
        const existing = localStore.getQuery(api.disputeItems.listByUser);
        if (existing) {
          localStore.setQuery(api.disputeItems.listByUser, undefined, existing.map(item =>
            item._id === args.disputeId ? { ...item, status: args.status } : item
          ));
        }
      }
    );
    ```

    **Loading and empty states:**
    - `items === undefined`: show a loading skeleton (3 placeholder cards) or "Loading disputes..." text
    - `items.length === 0`: show empty state — "No dispute items yet. Upload and analyze your credit reports to get started." with a link to `/upload`
    - `filteredItems.length === 0` (tabs active, no items for that bureau): "No items for this bureau."
