    **New endpoint** (per D-06):
    ```python
    @router.post("/{report_id}/analyze")
    async def analyze_report(report_id: str) -> AnalyzeResponse:
        """
        Analyze a parsed credit report with Claude AI.
        Fetches parsed data from Convex, calls Claude tool_use, validates FCRA citations,
        and returns dispute items ready to be stored.
        Idempotent: returns cached items if report already analyzed.
        """
    ```

    **Endpoint implementation steps:**
    1. Fetch parsed report from Convex via httpx using the same CONVEX_URL + CONVEX_API_KEY pattern already in reports.py. Call the Convex `creditReports:getReport` query with `reportId: report_id`.
    2. **Idempotency check (per Pitfall 4):** Query Convex for existing dispute_items via `disputeItems:listByReport` with `reportId: report_id`. If `len(existing_items) > 0`, return `AnalyzeResponse(dispute_items=existing_items, reused=True)` immediately — do NOT call Claude.
    3. Extract `parsed_data` from the report. If `parsed_data` is None or `parseStatus != "done"`, raise `HTTPException(status_code=422, detail="Report must be fully parsed before analysis")`.
    4. Deserialize `parsed_data` into a `ParsedReport` object: `report_obj = ParsedReport.model_validate(parsed_data)`.
    5. Call `dispute_items = await analyze_parsed_report(report_obj)`.
    6. Return `AnalyzeResponse(dispute_items=dispute_items, reused=False)`.
    7. Wrap the Claude call (steps 4-6) in try/except — on exception, log the error and raise `HTTPException(status_code=500, detail=str(e))`.

    **Important:** The endpoint returns the items but does NOT store them in Convex — that is done by the Convex action in Plan 03 which calls this endpoint. This separation keeps FastAPI stateless.
