    ```typescript
      onClose: () => void;
    }
    ```

    State:
    - `tab: "upload" | "manual"` — default "upload"
    - `file: File | null` — selected PDF
    - `uploading: boolean`
    - `outcome: "verified" | "deleted" | "corrected" | "no_response" | ""` — for manual entry
    - `notes: string` — optional manual notes (maps to reasonCode)
    - `error: string | null`
    - `success: boolean`

    Mutations/Actions:
    ```typescript
    const generateUploadUrl = useMutation(api.bureauResponses.generateResponseUploadUrl);
    const parseResponse = useAction(api.bureauResponses.parseResponse);
    const recordManual = useMutation(api.bureauResponses.recordResponseManual);
    ```

    **Upload tab handler:**
    1. Call `generateResponseUploadUrl()` to get upload URL
    2. PUT file to the URL with `Content-Type: application/pdf`
    3. Get storageId from PUT response JSON `{ storageId }`
    4. Call `parseResponse({ storageId, disputeItemId, bureau })`
    5. On success: set success=true, call onClose after 1.5s
    6. On error (including "unknown" outcome error from action): display error message and suggest manual entry tab

    **Manual entry handler:**
    1. Validate outcome is selected
    2. Call `recordResponseManual({ disputeItemId, bureau, outcome, reasonCode: notes || undefined })`
    3. On success: set success=true, call onClose

    **UI structure** (follow MarkAsSentDialog visual pattern):
    - Dialog.Root with open controlled externally (parent manages open state, passes onClose)
    - Two tab buttons: "Upload PDF" | "Enter Manually" — toggle tab state
    - Upload tab: file input `<input type="file" accept=".pdf">` + upload button + loading state
    - Manual tab: select dropdown for outcome, textarea for optional notes, submit button
    - Error message display in red below form
    - Success confirmation text in green
    - Outcome labels: verified → "Bureau Verified (Denied)", deleted → "Item Deleted (Win!)", corrected → "Item Corrected", no_response → "No Response Received"
