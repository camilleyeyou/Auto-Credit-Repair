    **saveResponse** (internalMutation) — called by the parseResponse action (Plan 02). Args:
    - disputeItemId: v.id("dispute_items")
    - userId: v.string()
    - bureau: bureau union
    - outcome: outcome union (verified | deleted | corrected | no_response | unknown)
    - accountName: v.optional(v.string())
    - responseDate: v.optional(v.number())
    - reasonCode: v.optional(v.string())
    - storageId: v.optional(v.id("_storage"))
    - entryMethod: v.union(v.literal("pdf_upload"), v.literal("manual"))

    Handler: insert into bureau_responses with recordedAt: Date.now(). Then update dispute_items.status using this mapping (only update if outcome !== "unknown" and outcome !== "no_response"):
    - "deleted" → "resolved"
    - "corrected" → "resolved"
    - "verified" → "denied"
    - "no_response" → leave status unchanged (stays "sent")
    - "unknown" → leave status unchanged

    **recordResponseManual** (public mutation) — RESP-03 manual entry path. Args:
    - disputeItemId: v.id("dispute_items")
    - bureau: bureau union
    - outcome: outcome union (verified | deleted | corrected | no_response)
    - accountName: v.optional(v.string())
    - responseDate: v.optional(v.number())
    - reasonCode: v.optional(v.string())

    Handler: verify auth, get identity.subject as userId, call ctx.db.insert("bureau_responses", { ...args, userId, recordedAt: Date.now(), entryMethod: "manual", storageId: undefined }). Then apply same status-update mapping as saveResponse.

    **getResponseForItem** (public query) — Args: { disputeItemId: v.id("dispute_items") }. Returns the single most-recent bureau_responses record for that item (order by recordedAt desc, take first) or null. Requires auth — verify userId matches.

    **getResponsesForUser** (public query) — Args: none. Returns all bureau_responses for the authenticated user, ordered by recordedAt desc. Used by the UI to show response history.
