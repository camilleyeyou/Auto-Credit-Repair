**getApprovedWithoutLetters** — internal query. Fetches all dispute items for the user with status "approved" that have no corresponding dispute_letters record. Implementation:
- Query dispute_items by_user index filtered to status === "approved"
- For each item, query dispute_letters by_dispute_item index
- Return only items where no letter exists yet

**saveLetter** — internal mutation. Inserts a dispute_letters record. Args: disputeItemId (v.id("dispute_items")), userId (v.string()), bureau (v.union of three literals), letterContent (v.string()), storageId (v.id("_storage")). Sets generatedAt to Date.now().

**generateLetters** — public action. Mirrors analyzeReport exactly. Handler:
1. `const identity = await ctx.auth.getUserIdentity(); if (!identity) throw new Error("Not authenticated");`
2. Fetch approved items without letters: `await ctx.runQuery(internal.letters.getApprovedWithoutLetters, { userId: identity.subject })`
3. Fetch user profile via internal query on users table: use `ctx.db.get()` pattern — create `getUserProfile` internalQuery that fetches by userId from the users table using the userId as a Convex Id. Note: identity.subject is the Convex user ID string — use `ctx.db.get(args.userId as Id<"users">)` inside the internalQuery.
4. Profile guard: if any of fullName, streetAddress, city, state, zip is missing/empty on the user doc, throw descriptive error: "Profile incomplete — add your name and mailing address in Profile before generating letters"
5. FASTAPI_URL guard: `const fastapiUrl = process.env.FASTAPI_URL; if (!fastapiUrl) throw new Error("FASTAPI_URL not set");`
6. Loop over approvedItems. For each item, wrap in try/catch (Pitfall 6 — one failure must not kill the batch):
   a. Build LetterRequest JSON body: map item fields (creditorName→creditor_name, accountNumberLast4→account_number_last4, disputeReason→dispute_reason, fcraSection→fcra_section, fcraSectionTitle→fcra_section_title) + compute fcra_section_usc from FCRA_USC dict (default to empty string if unknown) + user profile fields
   b. POST to `${fastapiUrl}/api/letters/generate`
   c. If !response.ok: log error and continue (skip this item)
   d. const result = await response.json() as { letter_html: string; pdf_base64: string }
   e. Decode base64 PDF: `const pdfBytes = Uint8Array.from(atob(result.pdf_base64), c => c.charCodeAt(0));`
   f. Store in Convex Storage: `const blob = new Blob([pdfBytes], { type: "application/pdf" }); const storageId = await ctx.storage.store(blob);`
   g. Insert dispute_letters record: `await ctx.runMutation(internal.letters.saveLetter, { disputeItemId: item._id, userId: identity.subject, bureau: item.bureau, letterContent: result.letter_html, storageId })`
   h. Update dispute item status: `await ctx.runMutation(internal.disputeItems.setLetterGenerated, { disputeId: item._id })`
   i. On catch: log `console.error("Letter generation failed for item", item._id, err)` and continue
7. Return void — no return value needed (frontend uses reactive query to see new letters)

**listByUser** — public query. Returns all dispute_letters for the authenticated user, ordered by generatedAt descending. Each returned document should include all fields from the dispute_letters table. Authentication required: `ctx.auth.getUserIdentity()`.

**getLetterDownloadUrl** — public query. Args: letterId (v.id("dispute_letters")). Gets the letter doc, verifies ownership (letter.userId === identity.subject), returns `await ctx.storage.getUrl(letter.storageId)`.

Important TypeScript note: import `Id` from "./_generated/dataModel" for the getUserProfile internalQuery cast.
